DYNAMIC DNS

	Le DDNS, ou Dynamic Domain Name System
	Pour rappel sur le fonctionnement du DNS, lorsque l'on rentre un nom de domaine dans l'URL de son navigateur, Le DNS est le service permettant de traduire ce nom de domaine son adresse IP associée. A titre d'exemple, lorsque l'on recherche une adresse particulière depuis son navigateur, le serveur DNS va chercher dans sa base de donnée une adresse IP appareillée à l'adresse web ; Une fois qu'il l'a trouvée, il va traduire le nom de domaine en l'adresse IP en question - comme youtube.com qui correspond à l'adresse IP 208.65.153.538 et google.com avec l'adresse IP 8.8.8.8.
	Dans une base de données DNS, un nom de domaine comme youtube.com est associée à une adresse IP en particulier. Cette adresse IP doit ne pas changer, il faut donc de ce fait qu'il s'agisse d'une adresse IP statique. En effet, si l'adresse IP de youtube venait à changer pour une nouvelle adresse, les utilisateurs ne seront plus en capacité d'accéder au site youtube.com, étant donné que le nom de domaine est toujours rattaché à l'ancienne adresse IP. Ainsi, dans le cas où l'adresse IP de youtube.com venait à changer, le DNS se chargera de mettre à jour sa base de donnée en associa le site avec la bonne adresse IP. L'intérêt de garder une adresse IP statique associée à un nom de domaine est que le DNS peut mettre jusqu'à 24h pour mettre à jour sa base de donnée avec la nouvelle IP. Cela signifierait donc que youtube.com perdra l'équivalent de 24h de business le temps que le DNS prenne en compte la bonne adresse IP. Prenons un cas de figure afin d'illustrer ce principe : on considère une entreprise depuis laquelle vous cherchez à accéder à votre machine personnelle d'adresse IP 21.70.19.80 chez vous, en passant par le service Microsoft Remote Desktop. Après avoir ouvert l'application, vous pourrez entrer l'adresse IP de la machine en question puis entrer le mot de passe et enfin valider : la connexion sera donc établie entre les deux appareils et vous pourrez vous contrôler la machine présente chez vous depuis votre entreprise. Toutefois, il est important de savoir que l'adresse IP de l'ordinateur de votre habitation changera de manière périodique, étant donné qu'elle n'est pas statique. Si vous n'êtes pas au courant, et que vous essayez une nouvelle fois d'établir une session de contrôle à distance avec la même adresse IP que précédemment, cela ne fonctionnera pas étant donné qu'il ne s'agit pas de la nouvelle adresse.
	C'est alors que le Dynamic DNS entre en jeu puisqu'il a pour mission de permettre l'accès à  l'ordinateur de votre foyer quand bien même votre adresse IP venait à changer et de créer un nom d'hôte personnalité comme "monpcperso.ddns.org" puis dans un second temps relier cet nom d'hôte à votre ordinateur. De ce fait, si l'adresse IP de votre habitation change, le DNS Dynamique actualisera la nouvelle adresse et le reliera à votre nom d'hôte personnalité automatiquement. De ce fait, l'objectif du DNS Dynamique est de s'assurer que vous aurez toujours accès à l'ordinateur de chez vous depuis votre poste d'entreprise.
	Si l'on revient à notre cas de figure précédent, au lieu d'indiquer l'adresse IP de votre appareil, vous n'aurez qu'à inscrire le nom d'hôte "monpcperso.ddns.org" personnalisé afin de se connecter à l'ordinateur en question. Peu importe le nombre de fois que l'adresse IP changera, cela n'aura pas de répercussion puisque la connexion à distance sera toujours reliée au nom d'hôte, lui-même mis à jour par le DDNS en cas de changement d'adresse IP.
