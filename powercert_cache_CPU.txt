CACHE CPU : QU'EST- CE QUE LA MÉMOIRE CACHE?

	Un ordinateur dispose de deux types mémoire : un premier type est utilisé grâce aux barrettes de RAM, soit la DRAM, qui constitue une mémoire utilisant des condensateurs pour stocker des données. Et ces condensateurs doivent, pour rappel, constamment être actualisés et alimentés en électricité lorsqu'il sont en fonctionnement étant donné qu'ils se vident en continu. Le second type de mémoire qu'un ordinateur utilise et qui n'a pas besoin d'être continuellement actualisé est appelée SRAM ou Static RAM ; C'est justement elle qui est employée dans le cache CPU. Etant donné qu'elle n'a pas besoin d'être sous tension en continu, elle est beaucoup plus rapide à utiliser que la DRAM mais coûte en contrepartie très cher.

	Le cache CPU désigne la mémoire interne du CPU et a pour mission de stocker des copies de données et d'instructions qui viennent de la RAM en l'attente d'être utilisées par le CPU. Plus généralement, le cache CPU conserve des données pour lesquelles il estime que le CPU utilisera encore et encore. En effet, lorsque le CPU doit accéder à un certain type de données, il vérifiera toujours en premier lieu si la mémoire cache CPU ne détient pas déjà la donnée en question. Dans le cas où le cache CPU ne possèderait pas ce que le CPU lui demande, il faudra donc passer par la mémoire primaire, ou la RAM, plus lente donc, pour récupérer les données dont il a besoin. C'est la raison pour laquelle la mémoire cache est si importante : puisque dans la mesure où le CPU accède à ce qu'il veut avec un cache donc plus rapidement, alors l'ordinateur sera bien plus rapide. L'objectif du cache est donc de rendre une machine plus efficace dans son exécution de tâches à un coût moins élevé.
	Un ordinateur pourrait techniquement fonctionner sans le cache CPU, mais il serait bien plus rend. En effet, bien que la RAM soit rapide, elle sera toujours trop lente pour alimenter en données le CPU. En effet, de nos jours les CPUs qui sont si rapides qu'il leur arrive de se mettre en attente, n'ayant rien à faire, toute en attendant que des données leurs parviennent : c'est ce qui crée des embouteillages, ou bottlenecks. De ce fait, c'est pour remédier à ce problème que le cache CPU a été implémenté ; Il agit en quelque sorte comme un intermédiaire entre le CPU et la RAM pour assister et alimenter le CPU des données dont il aurait besoin plus rapidement.

	Le cache CPU est décliné en 3 niveaux différentes :
 - Le niveau 1 du cache, appelé aussi primaire cache, ou cache primaire, est localisée directement sur le processeur ; De cette manière, il fonctionne à la même vitesse que celui-ci et est donc particulièrement rapide. Il s'agit du cache le plus rapide des 3 niveaux.
 - Le niveau 2 de cache, aussi appelé external cache, ou cache externe, a pour mission de récupérer des accès à des données récentes à partir du processeur et qui n'ont pas été intercepté par le cache de niveau 1. En résumé, si un CPU ne parvient pas à trouver les données dont il a besoin dans le cache de niveau 1, il va poursuivre ses recherches dans le cache de niveau 2.
 - Si le CPU n'arrive pas non plus à récupérer les données dans le cache de niveau 2, il va continuer de les rechercher dans le dernier niveau de cache, soit le cache de niveau 3. Celui-ci a pour mission, comme pour le cache 2 par rapport au niveau 1, de récupérer les accès de données qui n'ont pas pu être interceptées par le cache de niveau 2.
 - Dans le cas où le processeur n'est toujours pas parvenu à récupérer les données, il est alors contraint de se tourner vers la RAM plus lente pour trouver les donnes dont il a besoin.

	Par le passé, le cache de niveau 2 n'était placé dans une puce séparée sur la carte mère, mais dans les ordinateurs actuels, il est situé sur le processeur lui-même, comme pour le cache CPU de niveau 1. La puce de cache de niveau 2 est plus grande que celle de niveau 1, toutefois elle n'est pas aussi rapide. Le cache de niveau 3, comme pour les deux autres niveaux, est situé sur le CPU et est de plus grand taille que le cache de niveau 2 mais pas aussi rapide encore une fois. Le cache de niveau 3 est également désigné sous le nom de shared cache, ou cache partagé, puisque sa mémoire est partagée avec tous les coeurs du CPU, là où les caches de niveau 1 et 2 ne sont dédiés qu'à leur propre coeur attitré.
