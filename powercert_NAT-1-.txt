QU'EST CE QUE LE NAT ?

	NAT correspond à l'acronyme de Network Address Translation et il s'agit d'un service utilisé par les routeurs. Son rôle est de traduire un set d'adresses IP en un autre set afin d'aider à préserver le nombre limité d'adresses IPv4 en circulation dans le monde. En effet, lorsque les adresses IP version 4 ont été conçue, les ingénieurs n'avaient aucune idée qu'internet allait prendre autant d'ampleur. Bien qu'il y ait 4,294,967,296 adresses IPv4 publiques disponibles, les ingénieurs réseau pensaient qu'elles seraient en quantité suffisante, ce qui est largement démenti de nos jours. Ainsi, afin de limiter la pénurie d'adresses IP versions 4 publiques, des ingénieurs ont développé des adresses IP privées et la traduction d'adresses réseau, ou Network Address Translation (NAT).
	On distingue deux types d'adresses IPv4 différentes : les publiques, et les privées. Les adresses IP publiques sont répertoriées publiquement sur internet et vous avez besoin de les connaitre pour vous rendre sur internet. De l'autre côté, les adresses IP privées sont différentes puisqu'elles ne sont pas répertoriées publiquement. De ce fait, il n'est pas possible d'accéder directement à internet par le biais d'une adresses IP privée. Ces dernières ne sont utilisées que de manière interne dans un réseau local, donc dans une habitation ou un commerce, et non sur internet. Le rôle du routeur est d'attribuer aux appareils internes une adresses IP privée. Toutefois, la majorité des commerces ou des habitations ne nécessite que rarement une connexion internet pour un seul appareil, mais pour plusieurs. Ces appareils vont donc nécessiter une adresse IP publique s'ils souhaitent pouvoir accéder à internet. Dans ce cas, vous pourriez contacter votre fournisseur internet pour leur demander une adresse IP pour chacun de vos appareil ; Toutefois cela présente 3 inconvénients majeurs : cela couterait plus cher, ce serait inutile, et cela contribuerai au gaspillage d'adresses IP publiques. Si cette méthode avait été appliquée, nous aurions déjà été à assurément à court d'adresses IPv4. 
	Plutôt que de passer par cette méthode ci, nous pouvons demander à notre routeur d'attribuer lui-même les adresses IP privées pour les appareils de notre réseau local. Lorsque ceux-ci souhaiteront pouvoir accéder à internet, leur adresse IP privée sera traduite par le NAT dans le routeur en une adresse publique qu'on nous aura attribué. Bien entendu, le NAT ne fait pas que traduire des adresses IP privées en adresses IP publiques, mais également l'inverse, c'est-à-dire traduire des adresses IP publiques en adresses IP privées. En effet, si une machine à l'extérieur sur internet souhaite communiquer avec un ordinateur de son réseau local, l'adresse IP de cette machine devra etre traduite par le NAT pour l'ordinateur de destination.

	Toutefois, dans le futur, nous ne devrions plus avoir besoin de NAT ou d'adresses IPv4 privées grâce à la nouvelle génération d'adresses IP appelées IP version 6, ou IPv6. Grace à cette technologie, chaque unique appareil du globe détiendra sa propre adresse IP publique, il n'y aura donc pas besoin d'être traduite. La raison pour laquelle il ne sera pas nécessaire de les traduire, est que les IPv6 sont capable de fournir 340 undecilion d'adresses IP, soit environ 340*10^36 adresses IPv6 (340,282,366,920,938,463,463,374,607,431,768,211,456)
