32 VS 64 BITS

	Le hardware et le software d'un ordinateur peut être livré avec des versions 32 ou 64 bits et la différence entre les deux est la manière dont ils gèrent la mémoire. La taille de bit fait référence à la mémoire qu'on peut adresser : un système 32-bit peut référencer 2^32 bytes de mémoire, ce qui équivaut à 4 gigabytes, tandis qu'un système 64-bit peut référencer 2^64 bytes de mémoire, soit approximativement 16 exabytes, c'est-à-dire 4 milliards de fois plus de mémoire que le 32-bit. Le nombre est en réalité si élevé qu'il est virtuellement illimité, étant donné que nous n'utiliserons cette quantité de mémoire.

	Dans un ordinateur, afin que des données ou un programme puisse fonctionner, il doit au préalable être chargé dans la RAM. Les données sont donc stockées dans un disque dur plus lent puis chargée dans la RAM plus rapide, et une fois cette étape effectuée, le CPU peut accéder aux données en question et faire tourner le programme. Dans un système 32-bits, puisque le maximum de quantité de mémoire supportée est de 4 gigabytes, cela pourrait ne pas être suffisant pour contenir toutes les données dont le CPU a besoin, afin que l'ordinateur fonctionne aussi rapidement que possible. Lorsque cela se produit, une partie des données doit donc être conservée dans le disque dur pour compenser la faible mémoire. Ainsi, au lieu que les données aille de la RAM vers le CPU, elles doivent subir une étape supplémentaire en passant en premier temps du disque vers la RAM, enfin aller en direction du CPU.
	En revanche, dans le cas d'un système 64-bits, il est possible de stocker bien plus de mémoire que 4 Gigabytes, ce qui signifie que plus de données peuvent être stockées dans la RAM, plus rapide, que dans le disque dur, plus lent. Puisqu'il possible de stocker plus de données sur la RAM, l'ordinateur est donc en mesure de fonctionner bien plus rapidement. En résumé, c'est la raison pour laquelle un système 64-bits est plus rapide qu'un système 32-bits
